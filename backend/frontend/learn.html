Module 01

Java Basics for Automation

Learn the essential Java concepts you need to write Selenium automation scripts.

Why Java for Automation?

Java is one of the most widely used languages for Selenium automation. It has excellent community support, rich libraries, and integrates seamlessly with TestNG, Maven, and CI/CD tools.

Variables & Data Types

Java is strongly typed. You must declare a variable's type before using it.

Java 
// Primitive data types
int age = 25;
double price = 99.99;
boolean isActive = true;
char grade = 'A';

// Reference types
String name = "Selenium Learner";
String[] browsers = {"Chrome", "Firefox", "Edge"};

Object-Oriented Programming (OOP)

OOP is fundamental for the POM Design Pattern used in Selenium. Here are the core concepts:

  • Class & Object Blueprint and instance of that blueprint
  • Encapsulation Wrapping data with methods (getters/setters)
  • Inheritance Child class inherits from parent class
  • Polymorphism Same method, different behaviors
Java 
public class BasePage {
 protected WebDriver driver;

 public BasePage(WebDriver driver) {
 this.driver = driver;
 }

 public String getTitle() {
 return driver.getTitle();
 }
}

public class LoginPage extends BasePage {
 public LoginPage(WebDriver driver) {
 super(driver);
 }
}

Key Takeaway

Focus on Classes, Inheritance, and String methods these are the most used Java concepts in Selenium automation.

Control Flow

Control flow statements help you make decisions and loop through elements.

Java 
// If-else for conditional checks
if (driver.getTitle().contains("Login")) {
 System.out.println("On login page");
} else {
 System.out.println("Not on login page");
}

// For loop - iterate over elements
List<WebElement> links = driver.findElements(By.tagName("a"));
for (WebElement link : links) {
 System.out.println(link.getText());
}

Methods

Methods help you write reusable automation code. Every Selenium action can be wrapped in a method.

Java 
public void enterText(By locator, String text) {
 WebElement element = driver.findElement(locator);
 element.clear();
 element.sendKeys(text);
}

public String getElementText(By locator) {
 return driver.findElement(locator).getText();
}

public boolean isDisplayed(By locator) {
 return driver.findElement(locator).isDisplayed();
}
Module 02

Selenium WebDriver

Master the core Selenium WebDriver API for browser automation.

What is Selenium WebDriver?

Selenium WebDriver is a browser automation framework that allows you to control web browsers programmatically. It supports multiple browsers including Chrome, Firefox, Edge, and Safari.

Setting Up WebDriver

Java 
// Chrome WebDriver setup
WebDriverManager.chromedriver().setup();
WebDriver driver = new ChromeDriver();

// Navigate to URL
driver.get("https://your-practice-site.com");

// Maximize window
driver.manage().window().maximize();

// Set implicit wait
driver.manage().timeouts()
 .implicitlyWait(Duration.ofSeconds(10));

Locators Finding Elements

Selenium provides 8 locator strategies to find elements on a web page:

Java 
// By ID most reliable
driver.findElement(By.id("username"));

// By Name
driver.findElement(By.name("email"));

// By Class Name
driver.findElement(By.className("submit-btn"));

// By CSS Selector
driver.findElement(By.cssSelector("input[type='password']"));

// By XPath
driver.findElement(By.xpath("//button[@id='loginBtn']"));

// By Link Text
driver.findElement(By.linkText("Sign Up"));

// By Tag Name
driver.findElements(By.tagName("input"));

Best Practice

Use ID > CSS Selector > XPath as your locator priority. IDs are fastest and most reliable.

Common WebDriver Actions

Java 
// Type text into input field
driver.findElement(By.id("password")).sendKeys("developer2020");

// Click a button
driver.findElement(By.id("loginBtn")).click();

// Get text from element
String msg = driver.findElement(By.id("message")).getText();

// Clear input field
driver.findElement(By.id("search")).clear();

// Get attribute value
String val = driver.findElement(By.id("email")).getAttribute("value");

// Select dropdown
Select dropdown = new Select(driver.findElement(By.id("country")));
dropdown.selectByVisibleText("India");

WebDriver Waits

Waits are crucial in automation. They handle dynamic content that loads asynchronously.

Java 
// Explicit Wait wait for specific condition
WebDriverWait wait = new WebDriverWait(driver, Duration.ofSeconds(10));

// Wait until element is visible
WebElement elem = wait.until(
 ExpectedConditions.visibilityOfElementLocated(By.id("result"))
);

// Wait until element is clickable
wait.until(
 ExpectedConditions.elementToBeClickable(By.id("submitBtn"))
).click();

// Wait until text is present
wait.until(
 ExpectedConditions.textToBePresentInElementLocated(
 By.id("status"), "Success"
 )
);

Avoid Thread.sleep()

Never use Thread.sleep() in production automation. Use Explicit Waits (WebDriverWait) instead they're smarter and faster.

Module 03

TestNG Framework

Organize, run, and report your Selenium test cases with TestNG.

What is TestNG?

TestNG (Test Next Generation) is a testing framework inspired by JUnit. It's the most popular framework for Selenium because it offers annotations, assertions, parallel execution, and detailed reporting.

TestNG Annotations

Annotations control the order and behavior of your test methods:

Java 
public class LoginTests {

 WebDriver driver;

 @BeforeSuite
 public void beforeSuite() {
 System.out.println("Runs once before all tests");
 }

 @BeforeMethod
 public void setUp() {
 driver = new ChromeDriver();
 driver.manage().window().maximize();
 }

 @Test(priority = 1)
 public void testValidLogin() {
 driver.get("https://practice-site.com");
 // Test steps...
 Assert.assertEquals(driver.getTitle(), "Dashboard");
 }

 @Test(priority = 2)
 public void testInvalidLogin() {
 // Test with wrong credentials...
 Assert.assertTrue(errorMsg.isDisplayed());
 }

 @AfterMethod
 public void tearDown() {
 driver.quit();
 }
}

Common Assertions

Java 
// Verify exact match
Assert.assertEquals(actual, expected);

// Verify condition is true
Assert.assertTrue(element.isDisplayed());

// Verify condition is false
Assert.assertFalse(element.isSelected());

// Verify not null
Assert.assertNotNull(driver.findElement(By.id("result")));

TestNG XML Suite

Use XML suite files to organize and configure test execution:

XML 
<?xml version="1.0" encoding="UTF-8"?>
<suite name="AutomationSuite" parallel="tests" thread-count="2">
 <test name="LoginTests">
 <classes>
 <class name="com.tests.LoginTests" />
 </classes>
 </test>
 <test name="FormTests">
 <classes>
 <class name="com.tests.FormTests" />
 </classes>
 </test>
</suite>

Pro Tip

Use @DataProvider for data-driven testing run the same test with different inputs automatically.

What is Maven?

Maven is a build automation tool that manages project dependencies, builds, and project structure. It uses a pom.xml file to define everything your project needs.

Module 05

Extent Reports

Generate beautiful HTML test reports with screenshots and detailed logs.

What are Extent Reports?

Extent Reports is a reporting library that generates interactive, visually appealing HTML reports for your test results. It supports screenshots, logs, categories, and device information.

Setting Up Extent Reports

Java 
public class ExtentManager {
 static ExtentReports extent;
 static ExtentTest test;

 public static ExtentReports getInstance() {
 if (extent == null) {
 extent = new ExtentReports();
 ExtentSparkReporter spark =
 new ExtentSparkReporter("reports/TestReport.html");
 spark.config().setTheme(Theme.DARK);
 spark.config().setDocumentTitle("Automation Report");
 spark.config().setReportName("Test Results");
 extent.attachReporter(spark);
 }
 return extent;
 }
}

Using with TestNG Listener

Java 
public class TestListener implements ITestListener {

 @Override
 public void onTestSuccess(ITestResult result) {
 test.log(Status.PASS, "Test Passed: " + result.getName());
 }

 @Override
 public void onTestFailure(ITestResult result) {
 test.log(Status.FAIL, "Test Failed: " + result.getName());

 // Add screenshot on failure
 String screenshotPath = captureScreenshot(driver);
 test.addScreenCaptureFromPath(screenshotPath);
 }

 public String captureScreenshot(WebDriver driver) {
 TakesScreenshot ts = (TakesScreenshot) driver;
 File src = ts.getScreenshotAs(OutputType.FILE);
 String path = "screenshots/" + System.currentTimeMillis() + ".png";
 FileUtils.copyFile(src, new File(path));
 return path;
 }
}

Pro Tip

Always capture screenshots on test failure and attach them to Extent Reports. This makes debugging much easier!

Module 07

POM Design Pattern

Structure your automation project with the Page Object Model pattern.

What is Page Object Model (POM)?

POM is a design pattern that creates an Object Repository for web elements. Each web page has a corresponding Page Class that contains the web elements and methods for that page.

  • Separation of concerns Tests and page logic are separate
  • Reusability Page methods can be reused across tests
  • Maintainability Changes to UI only need updates in one place

POM Project Structure

Text 
src/
 main/java/
 pages/
 BasePage.java  Common methods
 LoginPage.java  Login page elements & actions
 HomePage.java  Home page elements & actions
 RegistrationPage.java
 utils/
 DriverFactory.java  WebDriver setup
 ConfigReader.java  Read config properties
 test/java/
 tests/
 BaseTest.java  @Before/@After setup
 LoginTest.java  Login test cases
 RegistrationTest.java

BasePage Class

Java 
public class BasePage {
 protected WebDriver driver;
 protected WebDriverWait wait;

 public BasePage(WebDriver driver) {
 this.driver = driver;
 this.wait = new WebDriverWait(driver, Duration.ofSeconds(10));
 }

 protected void click(By locator) {
 wait.until(ExpectedConditions.elementToBeClickable(locator)).click();
 }

 protected void type(By locator, String text) {
 WebElement el = wait.until(ExpectedConditions.visibilityOfElementLocated(locator));
 el.clear();
 el.sendKeys(text);
 }

 protected String getText(By locator) {
 return wait.until(ExpectedConditions.visibilityOfElementLocated(locator)).getText();
 }
}

Login Page Class

Java 
public class LoginPage extends BasePage {
 // Locators
 private By usernameField = By.id("username");
 private By passwordField = By.id("password");
 private By loginButton = By.id("loginBtn");
 private By errorMessage = By.id("error-msg");

 public LoginPage(WebDriver driver) {
 super(driver);
 }

 // Page Actions
 public void enterUsername(String username) {
 type(usernameField, username);
 }

 public void enterPassword(String password) {
 type(passwordField, password);
 }

 public HomePage clickLogin() {
 click(loginButton);
 return new HomePage(driver);
 }

 public String getErrorMessage() {
 return getText(errorMessage);
 }

 // Combine steps for convenience
 public HomePage loginAs(String user, String pass) {
 enterUsername(user);
 enterPassword(pass);
 return clickLogin();
 }
}

Test Class Using POM

Java 
public class LoginTest extends BaseTest {

 @Test
 public void testSuccessfulLogin() {
 LoginPage loginPage = new LoginPage(driver);
 HomePage homePage = loginPage.loginAs("admin", "pass123");

 Assert.assertEquals(
 homePage.getWelcomeMessage(),
 "Welcome, admin!"
 );
 }

 @Test
 public void testInvalidLogin() {
 LoginPage loginPage = new LoginPage(driver);
 loginPage.loginAs("wrong", "wrong");

 Assert.assertEquals(
 loginPage.getErrorMessage(),
 "Invalid credentials"
 );
 }
}

POM Benefits

If a web element's locator changes, you only update it in one place (the Page class). All tests using that page will automatically use the updated locator.
