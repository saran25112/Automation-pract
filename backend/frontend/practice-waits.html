Implicit Wait Setup

This demo loads automatically after 2 seconds. Use implicit wait so Selenium keeps trying to find the result element until it appears.

When to use it

Implicit wait is useful as a small global fallback for normal element lookup. Keep it short, and use explicit wait for dynamic or condition-based behavior.

View Selenium Code
Java Selenium 
// Apply implicit wait once after driver setup
driver.manage().timeouts().implicitlyWait(
 Duration.ofSeconds(5)
);

// Selenium will retry element lookup until the result appears
WebElement implicitResult = driver.findElement(
 By.id("implicitWaitResult")
);
System.out.println(implicitResult.getText());

// Best practice: use small implicit wait + explicit waits where needed

Dynamic Element Loader

Click to load an element after a 3-second delay. This is a good place to practice explicit wait with WebDriverWait.

View Selenium Code
Java Selenium 
// Click the load button
driver.findElement(By.id("loadElementBtn")).click();

// Wait for the dynamic element to appear
WebDriverWait wait = new WebDriverWait(driver, Duration.ofSeconds(10));
WebElement result = wait.until(
 ExpectedConditions.visibilityOfElementLocated(
 By.id("dynamicResult")
 )
);
System.out.println(result.getText());

Delayed Button

This button appears after a 5-second delay. Use explicit wait with elementToBeClickable.

View Selenium Code
Java Selenium 
// Click to start loading
driver.findElement(By.id("loadDelayedBtn")).click();

// Wait for delayed button to be clickable (max 10s)
WebDriverWait wait = new WebDriverWait(driver, Duration.ofSeconds(10));
WebElement delayedBtn = wait.until(
 ExpectedConditions.elementToBeClickable(
 By.id("delayedButton")
 )
);
delayedBtn.click();
