// Store current window handle
String mainWindow = driver.getWindowHandle();

// Click button to open new window
driver.findElement(By.id("openWindowBtn")).click();

// Get all window handles
Set<String> handles = driver.getWindowHandles();

// Switch to new window
for (String handle : handles) {
 if (!handle.equals(mainWindow)) {
 driver.switchTo().window(handle);
 break;
 }
}

// Perform actions on new window
System.out.println(driver.getTitle());

// Close new window and switch back
driver.close();
driver.switchTo().window(mainWindow);

// Get all rows from the table body
List<WebElement> rows = driver.findElements(
 By.cssSelector("#employeeTable tbody tr")
);

// Create a 2D array based on row count and 4 columns
String[][] tableData = new String[rows.size()][4];

// Use nested loops to read every cell
for (int i = 0; i < rows.size(); i++) {
 List<WebElement> cols = rows.get(i).findElements(By.tagName("td"));

 }
}

 System.out.print(tableData[i][j] + " | ");
 }
 System.out.println();
}

Close Browser

Learn the difference between close() and quit()

Close vs Quit

Understanding the difference between driver.close() and driver.quit() is essential for proper cleanup.

driver.close()

Closes only the current browser window/tab. Other windows remain open. Use this when working with multiple windows.

driver.quit()

Closes all browser windows and ends the WebDriver session completely. Always use this in @AfterMethod or @AfterTest.

View Selenium Code
Java Selenium 
// Close current window only
driver.close();

// Quit close all windows & end session
driver.quit();

// Best practice: in @AfterMethod
@AfterMethod
public void tearDown() {
 if (driver != null) {
 driver.quit();
 }
}
